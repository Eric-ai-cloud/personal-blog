## 📊 覆盖率检查

### 代码覆盖率

```bash
npm run test:coverage
```

**要求**:
- [ ] 语句覆盖率 > 80%
- [ ] 分支覆盖率 > 80%
- [ ] 函数覆盖率 > 80%
- [ ] 行覆盖率 > 80%

**核心模块要求**:
- [ ] 工具函数 > 90%
- [ ] 公共组件 > 85%
- [ ] API 层 > 85%

### 未覆盖代码分析

- [ ] 确认未覆盖的代码不需要测试（如类型定义）
- [ ] 或补充缺失的测试
- [ ] 或在代码中添加 /* istanbul ignore */ 并说明原因

## 🚀 最终确认

### 本地验证

```bash
# 1. 安装依赖
npm install

# 2. 运行测试
npm test

# 3. 检查覆盖率
npm run test:coverage

# 4. 构建验证
npm run build

# 5. 启动生产版本
npm start
```

### CI/CD 检查

- [ ] CI 流程配置正确
- [ ] 自动化测试运行
- [ ] 自动化部署配置
- [ ] 环境变量设置

### 最后确认

在提交 PR 前，请自问：

- [ ] 我愿意维护这段代码吗？
- [ ] 其他人能理解我的代码吗？
- [ ] 测试充分吗？
- [ ] 如果我是 reviewer，会批准吗？

**如果答案都是 "是"，那么可以提交了！** 🎉
